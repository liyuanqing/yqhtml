The four stages of Front-end development

Despite the growing complexity of Web applications, users of its front-end interface also put forward higher requirements, but today is still the front-end developers will not have much to think about in terms of front-end development of software engineering

Development efficiency to help the team or, worse, the front end also retains the "toy as simple as" stereotype, day after day, slash and burn.

The historic front-end development, always like stocking wild child, original predicament, can not help but lament!

In order to enhance its efficiency and operational performance of front-end development, front-end engineering construction teams generally go through three stages.

Library / frame selection

The first task is based on the front-end engineering construction project features technology selection.

Basically no one is completely started from 0 Site, even if it is Government websiteIt is normal to use a jquery,

JS/CSS Modular Development

Divide and conquer is the important thinking of software project, is the cornerstone of development and maintenance of complex systems, this also applies in the front-end development.

After resolving the basic development of efficient operation efficiency, front-end team to start thinking about maintenance efficiency, modular front-end is the most popular means of divide and conquer.

Modular development is not just code reuse, the greatest value should be modular development of divide and rule.

JS modular approach lot, AMD / CommonJS / UMD / ES6 Module, etc., corresponding to the frame and a lot of tools. CSS modular use less, sass

The terms that arrived the third stage belongs to the high-end, basically have the ability to develop and maintain general scale Web app.

The Front-end is less a technology problem, many engineering problems of software development.

We will face more problems, When we beginning a project such as:

  • 1. The general volume: multi-functional, multi-page, multi-state, multi-system;
  • 2. Large-scale: more than even the development team;
  • 3. High performance: CDN deployment, cache control, file fingerprint, cache reuse, merge request, demand loading, synchronous / asynchronous loading, moving end fold the embedded CSS, HTTP 2.0 server resource push.

These are undoubtedly serious of a series of systems engineering problems.

Speaking in front of three stages although many have compared to the "nascent" era of progress, but people develop and fine performance optimization support for the fourth phase seems to lack something.

component Development

Divide and conquer is indeed a very important means of engineering optimization.

As a GUI front-end software, the light JS / CSS modular enough for UI components partition also have the same urgent needs.

    Front-end component-based development concept

  • Each individual visual / interactive area on page 1 as a component;
  • 2. Each component corresponds to a project directory, the resources required components are in this directory nearest maintenance;
  • 3. Since the independence of the component, and therefore can be freely combined between components and assemblies;
  • 4. The page is just a container component, combination of components responsible for the formation of a fully functional interface;
  • 5. When a component does not need, or want to replace the components, you can delete the entire directory / replace.
